后续数据插入防护方案 - 完整指南

【问题总结】

之前出现的乱码问题原因:
1. MySQL JDBC 驱动不支持 utf8mb4 编码名称
2. 数据库连接字符编码配置不正确
3. HTTP 请求/响应没有明确指定 UTF-8 编码
4. 数据插入时没有进行字符编码验证

【解决方案架构】

已实现的多层防护:

第 1 层: 数据库连接层
├─ application.properties 配置
│  ├─ characterEncoding=utf8 (改为 Java 支持的标准编码)
│  ├─ useUnicode=true
│  └─ 其他 Hibernate 字符编码配置
└─ 数据库表字符集: utf8

第 2 层: HTTP 请求/响应层
├─ WebConfig (Web 配置类)
│  └─ 配置 StringHttpMessageConverter 使用 UTF-8
└─ CharacterEncodingFilter (字符编码过滤器)
   ├─ 设置请求编码为 UTF-8
   ├─ 设置响应编码为 UTF-8
   └─ 添加响应头确保浏览器正确解析

第 3 层: 业务逻辑层
├─ DroneController (控制器)
│  └─ 在保存前使用 CharacterEncodingUtil 验证和清理数据
└─ CharacterEncodingUtil (工具类)
   ├─ 验证字符串是否为有效 UTF-8
   ├─ 确保字符串使用 UTF-8 编码
   ├─ 检测字符串编码类型
   └─ 转换其他编码为 UTF-8

第 4 层: 数据库层
└─ 表字符集: utf8
   └─ 确保数据正确存储


【已创建的文件】

1. WebConfig.java
   位置: backend/src/main/java/com/powerinspection/config/WebConfig.java
   功能: 配置 HTTP 消息转换器使用 UTF-8 编码

2. CharacterEncodingFilter.java
   位置: backend/src/main/java/com/powerinspection/filter/CharacterEncodingFilter.java
   功能: 拦截所有 HTTP 请求/响应，确保使用 UTF-8 编码

3. CharacterEncodingUtil.java
   位置: backend/src/main/java/com/powerinspection/util/CharacterEncodingUtil.java
   功能: 提供字符编码验证和转换工具方法

4. DroneController.java (已更新)
   位置: backend/src/main/java/com/powerinspection/controller/DroneController.java
   功能: 在 POST 和 PUT 方法中使用工具类验证数据


【后续数据插入流程】

当添加新的无人机或其他数据时:

1. 前端发送请求
   ↓
   POST /api/drones
   Content-Type: application/json; charset=utf-8
   {
     "droneId": "UAV-05",
     "name": "无人机-05",
     "model": "DJI M300 RTK",
     ...
   }

2. CharacterEncodingFilter 拦截
   ↓
   设置请求编码为 UTF-8
   设置响应编码为 UTF-8

3. DroneController 处理
   ↓
   使用 CharacterEncodingUtil.ensureUtf8() 验证每个字符串字段
   确保所有中文字符正确编码

4. 数据保存到数据库
   ↓
   Hibernate 使用 UTF-8 编码保存数据
   数据库表使用 utf8 字符集存储

5. 返回响应
   ↓
   WebConfig 确保响应使用 UTF-8 编码
   前端正确显示中文字符

【常见问题排查】

问题 1: 新添加的数据仍然显示乱码

排查步骤:
1. 检查 application.properties 中的编码配置
2. 确认 WebConfig 和 CharacterEncodingFilter 已部署
3. 重启后端服务
4. 清空浏览器缓存
5. 检查数据库表字符集: SHOW CREATE TABLE drones;

解决方案:
```sql
ALTER TABLE drones CONVERT TO CHARACTER SET utf8 COLLATE utf8_general_ci;
```


问题 2: 通过 API 添加数据时出现错误

排查步骤:
1. 查看后端日志，找出具体错误信息
2. 检查前端请求头是否包含 charset=utf-8
3. 验证 JSON 数据格式是否正确
4. 检查字符串字段是否为 null

解决方案:
确保前端请求包含正确的 Content-Type:
```javascript
headers: {
  'Content-Type': 'application/json; charset=utf-8'
}
```


问题 3: 数据库连接失败

排查步骤:
1. 检查 MySQL 服务是否正常运行
2. 验证数据库连接字符串是否正确
3. 检查用户名和密码是否正确
4. 查看后端启动日志

解决方案:
确保 application.properties 中的数据库配置正确:
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/power_inspection?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=123456
```
